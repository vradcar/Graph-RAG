```
You are a graph entity resolver for an HVAC product knowledge graph.

Only return node IDs from the provided list - do not invent new ones.
```

5. **Dict `RELATION_QUERIES`** with four Cypher patterns using undirected `-[r:TYPE]-` (per D-04, Research pitfall 4):
   - `COMPATIBLE_WITH`: `MATCH (a {node_id: $node_id})-[r:COMPATIBLE_WITH]-(b) RETURN a.node_id AS src, 'COMPATIBLE_WITH' AS rel, b.node_id AS tgt`
   - `REPLACES`: same pattern with REPLACES
   - `SUPPORTS_WIRING`: same pattern with SUPPORTS_WIRING
   - `HAS_SPEC`: same pattern with HAS_SPEC

6. **Function `retrieve_graph_context(store: Neo4jGraphStore, node_ids: List[str], depth: int = 2) -> List[Tuple[str, str, str]]`** that:
   - For each node_id, runs all four RELATION_QUERIES via `store._driver.session()` and collects triples
   - Also calls `store.neighbors_multi_hop(node_id, depth=depth)` for BFS context
   - Deduplicates triples (use a set of tuples)
   - Returns list of (src, rel, tgt) tuples

7. **Function `graph_retrieve(store: Neo4jGraphStore, client: instructor.Instructor, model: str, question: str, known_ids: List[str], depth: int = 2) -> List[Tuple[str, str, str]]`** as the main entry point that:
   - Calls `resolve_entities` to get node_ids
   - Filters with `store.has_node(nid)` (per Research pitfall 2)
   - Logs filtered-out IDs to stderr
   - If no valid IDs, returns empty list
   - Calls `retrieve_graph_context` with validated IDs
   - Returns triples

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User question -> LLM | Untrusted NL input passed to entity resolver prompt |
| LLM response -> Cypher | LLM-returned node_ids used in parameterized Cypher queries |
| Neo4j -> LLM | Graph triples passed as context to answer generation prompt |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Injection | graph_retriever.py resolve_entities | mitigate | node_ids validated via `store.has_node()` before any Cypher execution; Cypher uses parameterized queries ($node_id), never string interpolation of user input |
| T-02-02 | Information Disclosure | generate.py | accept | Graph triples shown to user are the intended behavior; no PII in HVAC product data |
| T-02-03 | Spoofing | LLM entity resolver | accept | LLM may return wrong node_ids but they are filtered against known set; worst case is irrelevant results, not security breach |
</threat_model>
